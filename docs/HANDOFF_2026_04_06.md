## Current Blocker

## Ghidra Work Summary

### 1. Boss-Kill Phantom Return NOP
**Problem:** When host kills a boss, phantoms are silently sent home (no text, no effect).

**Call chain:**
```
FUN_14044ef30 (event dispatch, fires on boss death)
  -> CALL at exe+0x44ef7b (5 bytes: e8 30 2c d4 ff)
  -> FUN_140191bb0 (creates EventPhantomReturn objects)
  -> FUN_14018d660 (EventPhantomReturn constructor, vtable at 1410c33f8)
  -> Iterates player list, dispatches LeaveGuestPlayer per phantom
```

**Fix:** Runtime NOP of 5 bytes at exe+0x44ef7b via VirtualProtect. Verifies bytes match expected before patching.

**Code:** `PatchPhantomReturnOnBossKill()` in `src/sync/player_sync.cpp`

### 2. Player Cap 3 -> 6
**Problem:** DS2 hardcodes max 3 players, 4th player can't join.

**Call chain:**
```
FUN_1406ab050 (RequestNotifyJoinGuestPlayer message handler)
  exe+0x6ab0b6: c7 45 c3 03 00 00 00  MOV [RBP+local_6c], 0x3  <-- local var
  exe+0x6ab15b: c7 43 1c 03 00 00 00  MOV [RBX+0x1c], 0x3      <-- MaxPlayers in protobuf msg
```

**Fix:** Both `0x03` immediate bytes patched to `0x06` at runtime. Byte verification before patching.

**Code:** `PatchPlayerCap()` in `src/sync/player_sync.cpp`

**Caveat:** ds3os server may independently cap at 3. Untested whether client-side patch alone is sufficient.

### 3. Local Player Name Fix
**Problem:** `NSM → [+0x20] → +0x234` always returns the HOST's name. Joiners displayed as Sean's character name.

**Fix:** Switched primary path to `GMImp → [+0xA8] (GameDataManager) → +0x114`. Confirmed by Bob Edition CT + DS2S-META (`OFLD(ANYSOTFS, STRBASEA, 0xa8, 0x114)`).

**Code:** `ReadCharacterName()` in `src/sync/player_sync.cpp`, PATH 1 added before NSM fallback.

### 4. Duplicate Phantom Join Fix
**Problem:** HUD showed "1 player" with 3 people connected. Phantom names appeared twice in session manager.

**Root cause:** `OnPhantomJoined()` was wired to both `SerializeHook` (outgoing) AND `ParseHook` (incoming). Every join triggered twice.

**Fix:** Removed the call from `SerializeHook`. `ParseHook` only — both host and joiner see incoming messages there.

**Code:** `src/hooks/session_hooks.cpp`, SerializeHook `NotifyJoinGuestPlayer` block removed.

### 5. P2P Timeout Disabled During Seamless
**Problem:** Hamachi UDP heartbeats unreliable. Players would disappear from HUD after 60s.

**Fix:** `PeerManager::CheckTimeouts()` returns early when seamless mode is active. The ds3os session keeps everyone connected regardless.

**Code:** `src/network/peer_manager.cpp`

### 6. Diagnostic Name Scan Removed
Stripped the 15-path diagnostic scan from `ReadCharacterName()` now that the correct path is confirmed. Cleaner logs.

## Lessons Learned This Session
- Don't wire the same session event from both SerializeHook and ParseHook — double-fires
- Always verify expected bytes before writing an exe patch (protects against version mismatch)
- The "service not available" popup appears BEFORE the game attempts port 50031 — the Winsock hook can't prevent it
- ds3os server runs on multiple ports (50000/50005/50031 + 50060 range), not just one
- DS2's localhost check and offline mode behavior need more investigation
- Joiner ini needs to point at HOST's Hamachi IP, not 127.0.0.1
- Context pressure from long Ghidra image-analysis sessions may degrade subsequent task performance (hypothesis)
